# SubAgent-AGCR 规格

## 职责

从 diff 文件独立计算 AI 代码采纳率，与 observability 数据对比一致性。与 SubAgent-Intent 同时派发，互不依赖。

## 采纳率语义

- **one-shot 采纳率**：`agcr = retained_lines / final_lines`，其中 `retained_lines = one_shot_lines - removed_lines`。含义：最终交付代码中 AI 首次生成的占比。通常 ≤ 1.0。
- **one-shot 废弃率**：`abandonment_rate = removed_lines / one_shot_lines`。含义：AI 生成代码中被人工删除的比例。通常 ≤ 1.0。

## 主 Agent 传入参数

```json
{
  "repos": [
    {
      "repo": "{repo}",
      "base_to_one_shot_diff": "$OUTPUT_DIR/diffs/{repo}/{repo}-base-to-one-shot.diff",
      "one_shot_to_final_diff":  "$OUTPUT_DIR/diffs/{repo}/{repo}-one-shot-to-target-final.diff",
      "base_to_final_diff":     "$OUTPUT_DIR/diffs/{repo}/{repo}-base-to-target-final.diff"
    }
  ],
  "agcr_observed": "95.0%",
  "output_file": "$OUTPUT_DIR/agcr-calc.json"
}
```

## 行级过滤规则

统计 `+` 行时跳过：空行（strip 后为空）；注释行（以 `//`、`#`、`*`、`/*`、`*/` 开头）。

## 文件级过滤规则

通过 `filter_excluded_files.py` 的 `classify_file()` 执行，与 `split_hunks.py` 的 hunk 排除逻辑共用同一套规则，确保 AGCR 行数计算与归因 hunk 排除口径一致。排除规则按优先级顺序匹配（一旦命中即停止后续匹配）：

1. **doc_only** — 纯文档文件（.md/.txt/.rst），包括元析流程产物文件（work_status.md、design_interface.md、gate_test_analysis.md 等）和通用文档（changelog.md、readme.md 等）。JSON 元数据文件和 SQL 文件也归入此类
2. **test_file** — 测试文件（*Test.java / *Test.kt / src/test/ / *.test.js / *_test.go 等）
3. **auto_generated** — 自动生成代码，读取前 30 行 diff 内容检测标记（不区分大小写）：`@generated`、`do not edit`、`autogenerated by`、`code generated`、`auto-generated`、`generated by the protocol buffer compiler`、`generated by mapstruct`、`generated by protoc` 等。额外检测 Thrift Compiler 和 MapStruct Impl.java
4. **config_only** — 仅配置值变更（pom.xml 仅 version 标签变更 / application.yml 端口变更等）。若 diff 含 Spring 架构注解（@Bean、@Component、@Configuration 等）或 `<bean class=` 等 XML 结构声明则不排除
5. **whitespace** — 整个文件级别仅含空白符变化（去除所有空白后 added 和 removed 内容相同）
6. **auto_import** — 改动仅含 import / package / #include / use 语句

当 `filter_excluded_files.py` 不可用时（import 失败），回退为扩展名白名单 + auto-gen 标记检测：

扩展名白名单：`.java .kt .scala .py .js .ts .jsx .tsx .go .rs .c .cpp .h .cs .rb .php .swift .sh .bash .sql .xml .yaml .yml .properties .gradle .tf .proto .thrift`

auto-gen 标记检测：读取前 20 行 `+` 行拼接，包含 `@generated`、`do not edit`、`autogenerated by`、`code generated`、`auto-generated`、`generated by the protocol buffer compiler`、`generated by mapstruct` 则跳过。

## 计算步骤

1. 统计 base→one_shot 有效 `+` 行数 = `one_shot_lines`
2. 统计 one_shot→final 有效 `-` 行数 = `removed_lines`（钳制：removed ≤ one_shot）
3. 统计 base→final 有效 `+` 行数 = `final_lines`
4. 计算：`retained = one_shot - removed`，`agcr = retained / final`，`abandonment = removed / one_shot`
5. 与 observability 对比：误差 ≤ 5% → consistent；误差 > 5% → divergent

通过 `calc_agcr.py` 执行：

```bash
python3 "$SCRIPT_DIR/calc_agcr.py" \
  --repos-json $OUTPUT_DIR/repos-agcr-input.json \
  --agcr-observed {agcr_observed_value} \
  --output $OUTPUT_DIR/agcr-calc.json
```

## 输出文件

`$OUTPUT_DIR/agcr-calc.json`：

```json
{
  "agcr": 0.65,
  "abandonment_rate": 0.13,
  "agcr_consistency": "consistent|divergent",
  "agcr_observed": 0.67,
  "agcr_gap": 0.02,
  "per_repo": [
    {
      "repo": "repo-a",
      "one_shot_lines": 320,
      "removed_lines": 42,
      "retained_lines": 278,
      "final_lines": 428,
      "agcr": 0.65,
      "abandonment_rate": 0.13
    }
  ],
  "grand_total": {
    "one_shot_lines": 320,
    "removed_lines": 42,
    "retained_lines": 278,
    "final_lines": 428
  }
}
```

## 约束

- 必须从 diff 文件独立计算，禁止直接使用 `agcr_data_json.agcr_overall` 代替
- `removed_total` 不能超过 `one_shot_total`；超出时截断为 `one_shot_total`，abandonment_rate 设为 1.0
- `final_lines` 缺失时（base_to_final diff 不可用），agcr 设为 null，abandonment_rate 仍正常计算
- `one_shot_grand_total` 为 0 时，abandonment_rate 设为 null，不阻塞后续归因流程
